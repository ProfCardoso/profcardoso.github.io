# Projet Python : Site Web

## Contexte du projet

Par groupes de 2, 3 ou 4 élèves, vous devez concevoir et réaliser un **site web complet** sur un thème de votre choix.

Le site devra proposer :

- une **interface agréable et claire**;
- au moins **une page de contenu**;
- **un formulaire**;
- **un traitement des réponses** du formulaire;
- **l’utilisation de CSS** pour la mise en forme;
- **l’utilisation de JavaScript** pour rendre l’interface plus interactive.

## Consignes

Voici quelques consignes :

- Créez un dossier `Projet_3`, dans votre dossier `NSI`, et y ajouter un document texte avec les **nom**, les **prénom** et le **sujet** choisi par votre groupe.
- Ecrivez un document consignant : vos recherches sur le sujet, la répartition des tâches, et une explication du thème et des fonctionnalités de votre site.
- A la fin du projet, celui-ci sera à rendre complet (fichier(s) + document de réflexion) sur Pearltrees.

<span style="color:red">Attention ! L'IA peut être utilisé pour vous aider à chercher des idées ou vous aidez à écrire votre code, pas le faire à votre place !</span>

## Travail demandé

Votre site devra comporter au minimum :

### 1. Une page d’accueil

Elle présente :

- le nom du site
- son objectif
- des liens (pour aller sur le formulaire ou le/les page(s) de contenu)

### 2. Une page de contenu

Cette page doit apporter une vraie information à l’utilisateur. Elle devra avoir :

- des liens
- des images
- du texte

Elle peut contenir par exemple :

- un article
- une présentation
- une galerie
- un tableau d’informations
- une liste d’évènements
- une description d’activités ou de services

### 3. Un formulaire

Le formulaire doit permettre à l’utilisateur de saisir des informations, par exemple :

- inscription
- réservation
- demande d’informations
- contact
- avis
- questionnaire

Il devra contenir au moins 4 champs différents, par exemple :

- champ texte
- email
- liste déroulante
- boutons radio
- cases à cocher
- zone de texte

### 4. Un document traitant les réponses

Les données envoyées doivent être récupérées dans un fichier de traitement, par exemple en PHP.

Ce document doit :

- recevoir les données du formulaire
- vérifier qu’elles sont présentes
- afficher une réponse claire à l’utilisateur
- éventuellement enregistrer les réponses dans un fichier texte pour être réutilisées.

### 5. Autre documents nécéssaires

Le site doit utiliser un **fichier CSS externe** et un **fichier JavaScript externe**.

## Document de réponse

Voici les attendus pour le document de réponse à fournir

### Explications

Dans cette partie, vous devez :

- présenter le thème de votre site web,
- expliquer le but du site (à quoi sert-il ? pour qui ?),
- décrire les différentes pages réalisées,
- expliquer vos choix d’IHM (organisation, couleurs, navigation, lisibilité).

👉 On attend un texte structuré montrant que vous avez réfléchi à l’interface et à l’utilisateur.

### Intéractions

Dans cette partie, vous devez :

- décrire les fonctionnalités JavaScript mises en place,
- expliquer ce que fait chaque interaction,
- préciser en quoi cela améliore l’expérience utilisateur.

Exemples attendus :

- vérification de formulaire,
- affichage dynamique,
- changement de style,
- messages d’erreur ou de confirmation.

👉 Il ne suffit pas de dire “on a utilisé JavaScript”, il faut expliquer son rôle.

### Formulaire

Dans cette partie, vous devez :

- décrire le formulaire créé (champs présents),
- expliquer comment les données sont envoyées (GET ou POST),
- expliquer comment elles sont traitées en PHP,
- préciser si les données sont enregistrées (et comment),
- décrire le résultat affiché à l’utilisateur.

### Utilisations de l'IA

Dans cette partie, vous devez :

- expliquer comment vous avez utilisé l’IA (aide, correction, explication…),
- préciser ce que vous avez compris,
- indiquer ce que vous avez modifié ou adapté,
- montrer que vous êtes restés acteurs de votre projet.

👉 Il est important d’être honnête et de montrer un usage réfléchi. Les recherches faites sans IA ne sont pas à préciser.


### Difficultés rencontrées

Dans cette partie, vous devez :

- identifier les problèmes rencontrés (techniques ou organisationnels),
- expliquer pourquoi c’était difficile,
- décrire comment vous avez résolu ces difficultés.

Exemples :

- erreurs de code,
- compréhension du fonctionnement client/serveur,
- organisation du travail en groupe,
- mise en page CSS.

👉 On attend une réflexion sur votre travail, pas seulement une liste de problèmes.
